# Getting Started with Moca Desk

Moca Desk is the React frontend that renders forms, lists, dashboards, and reports from your MetaType definitions. It is distributed as the `@osama1998h/desk` npm package and scaffolded into every Moca project.

## Prerequisites

- **Node.js 22+** and **npm** (for desk development)
- **Go 1.26+** (for the `moca` CLI)
- An initialized Moca project (`moca.yaml` exists)

Node.js is required for development only. Production deployments serve pre-built static assets and do not need Node.js.

## Quick Start

```bash
# 1. Create a new Moca project (includes desk/ automatically)
moca init my-project
cd my-project

# 2. Install desk dependencies
moca desk install

# 3. Start the Vite dev server with hot module replacement
moca desk dev

# 4. Open http://localhost:3000/desk/ in your browser
```

For projects that need no frontend (headless API), pass `--skip-desk`:

```bash
moca init my-api --skip-desk
```

## Understanding the desk/ Directory

After `moca init`, your project contains:

```
my-project/
├── desk/
│   ├── package.json          # Depends on @osama1998h/desk, react, react-dom
│   ├── index.html            # SPA entry point
│   ├── vite.config.ts        # Uses mocaDeskPlugin() from @osama1998h/desk/vite
│   ├── tsconfig.json         # Composite project references
│   ├── tsconfig.app.json     # React app config
│   ├── tsconfig.node.json    # Node build tools config
│   ├── .moca-extensions.ts   # Auto-generated (do not edit, gitignored)
│   ├── .gitignore
│   └── src/
│       ├── main.tsx          # Entry point (~10 lines)
│       └── overrides/
│           ├── index.ts      # Project-level override entry point
│           └── theme.ts      # Theme/branding overrides
├── apps/
├── sites/
├── moca.yaml
└── ...
```

### Key files

**`desk/src/main.tsx`** is the entry point. It creates and mounts the desk application:

```tsx
import { createDeskApp } from "@osama1998h/desk";
import "../.moca-extensions";
import "./overrides";

const app = createDeskApp({
  // Project-level configuration
});

app.mount("#root");
```

**`desk/vite.config.ts`** uses the Moca Vite plugin that handles React, TailwindCSS, base path (`/desk/`), API proxy, and WebSocket proxy:

```ts
import { defineConfig } from "vite";
import { mocaDeskPlugin } from "@osama1998h/desk/vite";

export default defineConfig({
  plugins: [mocaDeskPlugin()],
});
```

**`desk/.moca-extensions.ts`** is auto-generated by `moca build desk`. It contains import and registration calls for all app desk extensions. Do not edit this file manually.

## @osama1998h/desk Package

The `@osama1998h/desk` package provides:

- **`createDeskApp(config?)`** -- factory function that initializes providers, router, and renders the desk UI
- **`mocaDeskPlugin(options?)`** -- Vite plugin that configures React, TailwindCSS, proxy, and base path
- **Registries** -- `registerFieldType()`, `registerPage()`, `registerSidebarItem()`, `registerDashboardWidget()` for extending the desk
- **Hooks** -- `useAuth()`, `useWebSocket()`, `useMetaType()`, `useDocument()`, `useDocList()`, `useRealtimeDoc()`, `useDocVersions()`, `useDeskConfig()`
- **Components** -- `FieldRenderer` for rendering document fields
- **Types** -- `FieldProps`, `MetaType`, `DocRecord`, `DeskConfig`, and more

The package is published to GitHub Packages (`npm.pkg.github.com`) and versions are synced with Go releases.

## CLI Commands

### `moca desk install`

Runs `npm install` in the `desk/` directory.

```bash
moca desk install           # Install dependencies
moca desk install --verbose # Show npm output
```

### `moca desk update`

Updates `@osama1998h/desk` to the latest compatible version and regenerates extension imports.

```bash
moca desk update           # Update + regenerate extensions
moca desk update --verbose # Show npm output
```

### `moca desk dev`

Starts the Vite development server with hot module replacement (HMR).

```bash
moca desk dev             # Start on default port (3000)
moca desk dev --port 4000 # Override port
```

The port is resolved in order: `--port` flag > `development.desk_port` in `moca.yaml` > default 3000.

### `moca build desk`

Discovers app extensions, generates `.moca-extensions.ts`, and runs a Vite production build.

```bash
moca build desk           # Production build
moca build desk --verbose # Show Vite build output
```

Output goes to `desk/dist/`, which `moca-server` serves as static files in production.

## Production Deployment

```bash
# On your CI/CD build machine:
cd my-project/desk
npm ci
moca build desk           # Produces desk/dist/

# Deploy desk/dist/ alongside Go binaries.
# moca-server serves desk/dist/ as static files -- no Node.js needed at runtime.
```

## Local Development with file: Protocol

When developing the `@osama1998h/desk` package itself (framework contributors), use npm's `file:` protocol to link the local desk source:

```json
{
  "dependencies": {
    "@osama1998h/desk": "file:../../desk"
  }
}
```

This creates a symlink so changes to the framework desk are immediately reflected.
